# 知识库

知识以“问题-方案-文献”的结构进行存储，以问题为核心。问题可由用户创建，或由Agent从论文精读报告中提炼，表现为一句精简但信息丰富的研究问题表述。

可对已有的论文精读报告进行问题、方案提炼，首先让Agent基于精读报告生成多个该文献希望解决的研究问题，要求具有一定通用性、层次高，而不是具体方法中的细节问题，然后利用向量相似度检索（基于faiss向量数据库）从数据库中检索相似的问题，如果可合并则合并，否则新增问题。然后，对这一文献解决这些问题的方案进行分别概括（基于论文精读报告），包含方法简述与效果和局限性。之后查看问题时，可以直接看到已有文献的方案以及文献链接。

该工作流在论文精读报告页面底部进行触发，一篇报告只能触发一次。
