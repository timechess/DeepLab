# 论文精读

对初筛得到的论文依次进行精读产出报告，该阶段从arxiv抓取论文pdf到本地，url为https://arxiv.org/pdf/{id}，此处id填入论文id。

抓取论文到本地临时文件夹后，先通过 **Mistral OCR** 将 PDF 转成 Markdown 文本，再将 OCR 文本插入到提示词中，交由 google-genai 模型生成精读结果。精读过程分两步：

1. 着重提问，带着问题进行阅读，该阶段Agent需要首先给出全文的简要概括，包括该论文针对什么问题，提出了怎样的解决方案，主要贡献和创新点有哪些，并给出全文的章节大纲，然后结合自己的思考提出多个相关的研究问题。
2. 新的Agent接收上一步输出的内容后，继续阅读论文撰写精读报告，报告格式要求分为如下章节：1）全文概括，即上一步中的简要概括。2）方法细节，给出论文提出方法的可操作细节，按实操顺序（最好举例说明）介绍一遍。3）实验分析，给出实验部分的概要，包括进行了哪些实验以及分析性结论。4）对研究问题的回答，即对上一步提出的研究问题进行回答。5）论文评价，以审稿人的身份给出该论文的优劣评价。6）扩展阅读，给出相关文献阅读清单。

上述步骤均以 OCR 后的论文文本作为上下文输入，两个 Agent 的角色定位 prompt 不同。

实现约束：

1. 本地 PDF 保存在临时目录，OCR 完成后即删除。
2. 上传到 Mistral 的文件在 OCR 完成后即调用删除接口清理（best effort）。
3. 数据库日志中不保存论文正文文本，只记录论文 ID、页数、字符数等元信息；但传给大模型的 prompt 必须包含 OCR 全文。

环境变量：

1. `MISTRAL_API_KEY`（或 `mistral_api_key`）：Mistral OCR Key。
2. `MISTRAL_BASE_URL`（可选）：默认 `https://api.mistral.ai`。
3. `MISTRAL_OCR_MODEL`（可选）：默认 `mistral-ocr-latest`。

论文精读报告生成完毕后，存入数据库时还要带一个 `comment` 字段用于存储用户评论，初始为空，用户可针对报告给出评价或笔记。
